32비트 소수 생성 가속화:
알고리즘 및 구현 상세 리포트 (1부)

프로젝트 서문 및 사양

본 문서는 Linux 환경에서 32비트 부호 없는 정수(uint32_t) 범위 내의 모든 소수를 가장 빠른 속도로 생성하기 위한 C 프로그래밍 여정을 기록합니다. 생성된 소수는 PRIMES라는 이진 파일에 리틀 엔디안 방식으로 기록되어야 합니다.

  • 데이터 포맷: 4바이트당 소수 1개 저장 (예: 02 00 00 00 03 00 00 00...)
  • 무결성 검증: SHA-256 해시 272eb05aa040ba1cf37d94717998cbbae53cd669093c9fa4eb8a584295156e15
  • 하드웨어 제약: 메모리 사용량 1GB 이내 엄수.
  • 정확성 원칙: 2보다 큰 어떤 수도 사전에 검증하지 않고 소수로 간주해서는 안 됨.

1단계: 시행 나눗셈 (Trial Division)

가장 기본이 되는 알고리즘으로, 수 $n$이 $\sqrt{n}$ 이하의 소수들로 나누어 떨어지는지 확인합니다.

// n이 소수인지 확인합니다.
// primes: 2부터 n의 제곱근까지 순서대로 나열된 소수 배열
bool is_prime(uint32_t n, uint32_t* primes) {
	for (size_t i=0; ; i++) {
		uint32_t p = primes[i];
		if (n % p == 0) return false;
		// p*p 오버플로우 방지 및 소수 판정 종료
		if (p >= 0xFFFF || p*p >= n) return true;
	}
}

이 방법의 시간 복잡도는 $O(n\sqrt{n}/\log n)$이며, 전체 $2^{32}-1$ 범위를 탐색하기 위해 약 203,739,216개의 공간을 할당합니다. (실제 소수 개수는 203,280,221개이나, Dusart의 경계를 사용하여 안전하게 공간을 확보함)

  • 실행 시간: 약 24분 20초 소요.
2단계: 휠 인수분해 (Wheel Factorization)

명백한 합성수(2, 3, 5의 배수 등)를 사전에 제거하여 확인해야 할 후보 숫자를 줄입니다.

  • 원리: 30($2 \times 3 \times 5$)을 법으로 하여 30과 서로소인 숫자들($\{1, 7, 11, 13, 17, 19, 23, 29\}$)만 검사합니다.
  • 효과: 전체 자연수의 약 26.6%(또는 휠 크기에 따라 약 20%)만 소수 후보군으로 선정됩니다.
// 서로소 기반 휠 순환 알고리즘
uint32_t last = wheel_last(wheel);
uint32_t turn = 0;
while (true) {
	for (size_t i=0; i < wheel.n_spokes; i++) {
		uint32_t n = wheel.size * turn + wheel.coprime_spokes[i];
		if (is_prime(n, primes)) {
			primes[p_idx++] = n;
		}
		if (n == last) goto exit_loop;
	}
	turn++;
}
  • 실행 시간: 약 23분 30초 소요.
  • 평가: 검사 대상은 80% 가량 줄었으나, 제거된 숫자들은 원래 시행 나눗셈에서 가장 빠르게 걸러지는 작은 소수의 배수들이었기에 실제 속도 향상은 미미함.
3단계: 에라토스테네스의 체 (Sieve of Eratosthenes)

개별 검증 대신, 목록에서 합성수를 지워나가는 방식을 취합니다. 시간 복잡도는 $O(n \log \log n)$으로 가장 효율적입니다.

비트 배열(Bit Array) 최적화

4.3GB가 필요한 불리언 배열 대신, 각 비트에 상태를 저장하여 메모리 사용량을 **537MB**로 단축했습니다.

struct BitArray list = bitarray_init((size_t)0xFFFFFFFF);
for (uint32_t p = 2; p <= 0xFFFF && p*p <= 0xFFFFFFFF; p++) {
	if (bitarray_get(list, p)) continue;
	// p의 자기 자신을 제외한 모든 배수 소거 (p-sifting)
	for (size_t i=2; i <= 0xFFFFFFFF / p; i++) {
		bitarray_set(&list, i*p, true);
	}
}
  • 실행 시간: 약 32초 소요. (이전 방식 대비 약 40배 이상 향상)
  • 메모리 총량: 비트 배열(537MB) + 소수 결과 배열(815MB) $\approx$ 1.3GB (약간 초과이나 유연한 허용치 이내)
